# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

# Данные
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}


Интерфейс IProduct описывает структуру данных для представления товара в системе (например, в интернет‑магазине). Он задаёт набор полей, которые должны быть у любого объекта товара, обеспечивая единообразие данных и типобезопасность в TypeScript‑коде.
Поля интерфейса и их значение

id: string
Тип: string (строка).
Назначение: уникальный идентификатор товара.
Значение: служит для однозначной идентификации товара в системе. Используется для: поиска товара (getProductById);отслеживания наличия в корзине (hasItem);связывания данных (например, позиций в заказе с каталогом);формирования URL‑адресов детальных страниц (/product/:id).

description: string
Тип: string.
Назначение: подробное описание товара.
Значение: содержит развёрнутую информацию о товаре для покупателя: характеристики, особенности, способы применения. Отображается на детальной странице товара.

image: string
Тип: string.
Назначение: ссылка (URL) на изображение товара.
Значение: указывает путь к файлу изображения (например, https://example.com/images/product-123.jpg). Используется для отображения визуальной информации о товаре в каталоге, карточках товаров и корзине.

title: string
Тип: string.
Назначение: название (заголовок) товара.
Значение: краткое и понятное наименование товара, которое видит пользователь в каталоге, корзине, фильтрах. Должно быть информативным и лаконичным.

category: string
Тип: string.
Назначение: категория, к которой относится товар.
Значение: классифицирует товар для организации каталога (например, «Электроника», «Книги», «Одежда»). Используется для:фильтрации и группировки товаров;построения навигации по сайту;реализации поиска по категориям.

price: number | null
Тип: number | null (число или null).
Назначение: цена товара.

Значение:числовое значение (например, 1999.99) — текущая цена товара в денежных единицах;null — означает, что цена на товар в данный момент не установлена или недоступна. Это позволяет корректно обрабатывать:товары под заказ;позиции, временно снятые с продажи;случаи, когда цена уточняется у менеджера.

Интерфейс IProduct:задаёт контракт для объектов товара — любой товар в системе должен соответствовать этой структуре;обеспечивает типобезопасность — компилятор TypeScript проверит, что все обязательные поля присутствуют и имеют правильный тип;упрощает взаимодействие между компонентами приложения (каталог, корзина, оформление заказа) за счёт единого формата данных;позволяет гибко обрабатывать особые случаи (например, отсутствующую цену через null).





type TPayment = 'card' | 'cash' | ''
interface IBuyer {
  payment: TPayment;
  email: string;
  phone: string;
  address: string;
}


Интерфейс IBuyer описывает структуру данных для представления информации о покупателе в системе (например, в интернет‑магазине). Он задаёт набор полей, которые должны быть у любого объекта покупателя, обеспечивая единообразие данных и типобезопасность в TypeScript‑коде.
Поля интерфейса и их значение:

payment: TPayment
Тип: TPayment.
Назначение: хранит выбранный покупателем способ оплаты заказа.
Значение: может принимать одно из трёх значений:'card' — покупатель выбрал оплату картой;'cash' — покупатель выбрал наличный расчёт;'' — способ оплаты пока не выбран (например, на раннем этапе оформления заказа).
Использование: используется при формировании заказа, отображении информации о способе оплаты в профиле пользователя или в истории заказов.

email: string
Тип: string.
Назначение: электронная почта покупателя.
Значение: адрес электронной почты для связи с покупателем. Используется для:отправки уведомлений о статусе заказа;доставки электронных чеков и гарантийных талонов;рассылки промо‑материалов и новостей магазина (при согласии пользователя);восстановления пароля и подтверждения учётной записи.

phone: string
Тип: string.
Назначение: номер телефона покупателя.
Значение: контактный номер для оперативной связи. Используется для:подтверждения заказа по телефону;информирования о доставке;решения срочных вопросов, связанных с заказом;двухфакторной аутентификации (если поддерживается).

address: string
Тип: string.
Назначение: адрес доставки заказа.
Значение: полный адрес, куда необходимо доставить товар. Может включать:город, улицу, дом, квартиру;индекс;дополнительные указания (например, «подъезд 2, домофон 123»).
Использование: передаётся службе доставки для организации доставки товара.


Интерфейс IBuyer:задаёт контракт для объектов покупателя — любой покупатель в системе должен соответствовать этой структуре;
обеспечивает типобезопасность — компилятор TypeScript проверит, что все обязательные поля присутствуют и имеют правильный тип;
упрощает взаимодействие между компонентами приложения (оформление заказа, профиль пользователя, история заказов) за счёт единого формата данных;
использует ограниченный набор значений для способа оплаты (TPayment), что снижает вероятность ошибок ввода;
позволяет корректно обрабатывать незаполненные данные (например, пустой payment) без нарушения работы системы.




## Модели данных

Класс ProductCatalog
Назначение и зона ответственности
Класс ProductCatalog предназначен для управления коллекцией товаров в приложении (например, в интернет‑магазине или каталоге продукции). Его основная задача — централизованно хранить данные о товарах и предоставлять механизмы для работы с ними.
Зона ответственности класса включает:
хранение полного списка доступных товаров;
отслеживание текущего выбранного товара для детального просмотра;
загрузку данных о товарах в каталог;
предоставление доступа ко всем товарам или к конкретному товару по идентификатору;
установку и получение информации о выбранном товаре.
Конструктор класса
typescript
constructor() {}
Конструктор не принимает параметров. При создании экземпляра класса:
поле products инициализируется пустым массивом ([]);
поле selectedProduct инициализируется значением null.
Поля класса
products: IProduct[] = []
Тип данных: IProduct[] (массив объектов, соответствующих интерфейсу IProduct).
Что хранит: полный список всех товаров, доступных в каталоге. Каждый элемент массива — объект товара с полями id, title, price и т. д.
За что отвечает:
хранение данных о товарах;
обеспечение доступа к каталогу для отображения, фильтрации, поиска;
предоставление исходных данных для методов класса (например, getProductById).
selectedProduct: IProduct | null = null
Тип данных: IProduct | null (объект товара по интерфейсу IProduct либо null).
Что хранит: ссылку на товар, который в данный момент выбран пользователем для подробного просмотра (например, после клика на карточке товара в каталоге).
За что отвечает:
сохранение состояния «текущего товара»;
предоставление данных для детальной страницы товара (getProduct);
изначально имеет значение null, что означает, что ни один товар не выбран.
Методы класса
setItems(items: IProduct[]): void
Параметры: items: IProduct[] — массив объектов товаров, которые необходимо загрузить в каталог.
Возвращаемое значение: void (ничего не возвращает).
Назначение: заменяет текущий массив товаров (products) на новый, переданный в параметре. Используется, например, после получения данных с сервера.
getItems(): IProduct[]
Параметры: отсутствуют.
Возвращаемое значение: IProduct[] — полный массив всех товаров, хранящихся в каталоге.
Назначение: предоставляет доступ к полному списку товаров. Может использоваться для отображения каталога на странице, передачи данных в другие компоненты или модули.
getProductById(id: string): IProduct | null
Параметры: id: string — уникальный идентификатор товара, который необходимо найти в каталоге.
Возвращаемое значение:
IProduct — найденный товар, если он существует в массиве products;
null — если товар с указанным id не найден.
Назначение: выполняет поиск товара по его уникальному идентификатору. Используется, например, при переходе по ссылке вида /product/123 для загрузки данных о конкретном товаре.
setProduct(selectedProduct: IProduct): void
Параметры: selectedProduct: IProduct — объект товара, который нужно установить как текущий выбранный для подробного отображения.
Возвращаемое значение: void.
Назначение: сохраняет переданный товар в поле selectedProduct. После вызова этого метода метод getProduct() будет возвращать именно этот товар.
getProduct(): IProduct | null
Параметры: отсутствуют.
Возвращаемое значение:
IProduct — текущий выбранный товар (если он был установлен методом setProduct);
null — если ни один товар не выбран (поле selectedProduct имеет значение null).
Назначение: возвращает товар, который был сохранён методом setProduct(). Используется для получения данных о товаре, который нужно показать в детальном представлении (например, на странице товара).

Класс ShoppingCart
Назначение и зона ответственности
Класс ShoppingCart предназначен для управления корзиной покупок в интернет‑магазине или аналогичном приложении. Его основная задача — хранить и обрабатывать данные о товарах, выбранных покупателем для приобретения.
Зона ответственности класса включает:
хранение списка товаров в корзине;
добавление и удаление товаров из корзины;
очистку корзины;
расчёт общей стоимости товаров;
подсчёт количества товаров в корзине;
проверку наличия товара в корзине по идентификатору.
Конструктор класса
typescript
constructor() {}
Конструктор не принимает параметров. При создании экземпляра класса поле items инициализируется пустым массивом ([]).
Поля класса
items: IProduct[] = []
Тип данных: IProduct[] (массив объектов, соответствующих интерфейсу IProduct).
Что хранит: список товаров, добавленных пользователем в корзину. Каждый элемент массива — объект товара с полями id, title, price и т. д.
За что отвечает:
хранение данных о выбранных товарах;
обеспечение доступа к содержимому корзины для отображения и расчётов;
предоставление исходных данных для методов класса (например, getPrice, getCount).
Методы класса
getItems(): IProduct[]
Параметры: отсутствуют.
Возвращаемое значение: IProduct[] — массив всех товаров, находящихся в корзине в текущий момент.
Назначение: предоставляет полный доступ к содержимому корзины. Используется для отображения списка товаров на странице корзины или в мини‑превью.
addItem(item: IProduct): void
Параметры: item: IProduct — объект товара, который необходимо добавить в корзину.
Возвращаемое значение: void (ничего не возвращает).
Назначение: добавляет переданный товар в массив items. Если товар уже присутствует в корзине, он будет добавлен повторно (учитывается количество).
deleteItem(item: IProduct): void
Параметры: item: IProduct — объект товара, который нужно удалить из корзины. Удаление выполняется по уникальному идентификатору товара (id).
Возвращаемое значение: void.
Назначение: удаляет указанный товар из массива items
clear(): void
Параметры: отсутствуют.
Возвращаемое значение: void.
Назначение: полностью очищает корзину — устанавливает массив items в пустой массив ([]). Используется, например, после успешного оформления заказа.
getPrice(): number
Параметры: отсутствуют.
Возвращаемое значение: number — общая стоимость всех товаров в корзине (в денежных единицах).
Назначение: вычисляет сумму цен всех товаров в корзине. Учитывает только товары, у которых поле price не равно null.
getCount(): number
Параметры: отсутствуют.
Возвращаемое значение: number — общее количество товаров в корзине.
Назначение: возвращает количество товаров в корзине как число элементов в массиве items. Это не количество уникальных позиций, а общее число товаров (если один товар добавлен дважды, он учитывается дважды).
hasItem(id: string): boolean
Параметры: id: string — уникальный идентификатор товара, наличие которого нужно проверить в корзине.
Возвращаемое значение: boolean:
true — если товар с указанным id присутствует в корзине хотя бы в одном экземпляре;
false — если товара с таким id нет в корзине.
Назначение: проверяет, есть ли в корзине товар с заданным идентификатором. Используется, например, чтобы изменить кнопку «Добавить в корзину» на «Уже в корзине».

Класс Buyer
Назначение и зона ответственности
Класс Buyer предназначен для хранения и управления данными о покупателе в системе (например, в интернет‑магазине). Его основная задача — обеспечивать:
безопасное хранение контактной информации, адреса доставки и способа оплаты;
гибкое обновление данных покупателя;
получение актуальных данных о покупателе;
очистку данных (например, после успешного оформления заказа);
валидацию обязательных полей с указанием конкретных ошибок.
Зона ответственности класса включает:
хранение персональных данных покупателя;
обновление данных по частям (без перезаписи всего объекта);
предоставление полной информации о покупателе в виде объекта;
сброс всех данных до начального состояния;
проверку заполнения обязательных полей и формирование сообщений об ошибках.
Конструктор класса
typescript
constructor() {}
Конструктор не принимает параметров. При создании экземпляра класса все поля инициализируются пустыми строками ('').
Поля класса
payment: TPayment = ''
Тип данных: TPayment ('card' | 'cash' | '').
Что хранит: выбранный покупателем способ оплаты.
За что отвечает: содержит информацию о способе оплаты, которая будет использована при оформлении заказа. Допустимые значения:
'card' — оплата банковской картой;
'cash' — наличный расчёт;
'' — способ оплаты не выбран (начальное состояние).
email: string = ''
Тип данных: string.
Что хранит: адрес электронной почты покупателя.
За что отвечает: обеспечивает канал электронной коммуникации с покупателем:
отправка уведомлений о статусе заказа;
доставка электронных чеков и гарантийных талонов;
восстановление пароля и подтверждение учётной записи.
phone: string = ''
Тип данных: string.
Что хранит: номер телефона покупателя.
За что отвечает: предоставляет контактный номер для оперативной связи:
подтверждение заказа по телефону;
информирование о доставке;
решение срочных вопросов, связанных с заказом;
двухфакторная аутентификация (если поддерживается).
address: string = ''
Тип данных: string.
Что хранит: полный адрес доставки заказа (город, улица, дом, квартира, индекс, дополнительные указания).
За что отвечает: передаёт службе доставки точные данные для организации доставки товара.
Методы класса
setBuyerData(data: Partial<IBuyer>): void
Параметры: data: Partial<IBuyer> — объект с полями для обновления. Может содержать любое подмножество полей интерфейса IBuyer.
Возвращаемое значение: void (ничего не возвращает).
Назначение: обновляет указанные поля объекта, не затрагивая остальные. Позволяет гибко изменять данные (например, обновить только email, оставив остальные поля без изменений).
getData(): IBuyer
Параметры: отсутствуют.
Возвращаемое значение: IBuyer — объект, содержащий актуальные данные покупателя (все поля класса).
Назначение: возвращает полный набор данных покупателя в виде объекта типа IBuyer. Используется для:
передачи данных в другие компоненты или модули;
сериализации данных (например, для сохранения в localStorage);
отображения информации в профиле пользователя.
clearData(): void
Параметры: отсутствуют.
Возвращаемое значение: void.
Назначение: очищает все данные покупателя — устанавливает все поля в пустые строки (''). Используется после:
успешного оформления заказа;
сброса формы ввода данных;
выхода пользователя из системы.
validate(): ValidationErrors
Параметры: отсутствуют.
Возвращаемое значение: ValidationErrors — объект с сообщениями об ошибках для каждого поля.
Тип ValidationErrors:
typescript
type ValidationErrors = {
  payment: string;
  email: string;
  phone: string;
  address: string;
};
Назначение: проверяет, заполнены ли обязательные поля покупателя. Для каждого незаполненного поля формирует сообщение об ошибке.
Логика работы:
если поле payment пустое, в obj.payment записывается сообщение об ошибке;
аналогично проверяются email, phone и address;
возвращается объект obj с сообщениями (пустые строки означают отсутствие ошибки для соответствующего поля).
Использование: позволяет отобразить пользователю конкретные ошибки ввода перед отправкой данных на сервер.





### Слой коммуникации
Класс: Api
Назначение: класс‑адаптер для взаимодействия с сервером «веб‑ларёк».
Принцип работы: использует композицию — получает экземпляр класса Api в конструкторе и делегирует ему выполнение HTTP‑запросов.
Зависимости:
интерфейс IApi (из types/index.ts) — контракт для HTTP‑клиента;
существующие типы данных (IProduct, IBuyer и др.) — для типизации запросов и ответов.
Методы:
getProducts(): Promise<IProduct[]> — выполняет GET‑запрос к /product/, возвращает массив товаров.
sendOrder(orderData: IOrderRequest): Promise<IOrderResponse> — выполняет POST‑запрос к /order/, отправляет данные заказа, возвращает подтверждение.
